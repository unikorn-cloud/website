Privacy Statement

This is not a managed service, it is a FOSS application, so we do not, and will not, as an entity gather any information about you or your users.

Any informantion that is gatherered about your users e.g. Google Groups, is used sorely for functionality purposes e.g. management of RBAC, that are not possible through existing oauth2/OIDC standards.

Use of Identity Provider Data

Information is ephemeral where possible, only what is absolutely needed to function is ever stored. No information is never transmitted to any 3rd party.

OpenID Connect

Email addresses provide a canonical identifier for a user. These are used to grant access to resources via RBAC, and provide audit logging. Profile data e.g. forename, surname, picture may be used to personalize UX.

Groups

Groups are used to provide access to resources via RBAC, and do so by implicitly mapping a user to internal groups and the roles associated with them.

Only what is required to function is ever stored e.g. group identifiers.
